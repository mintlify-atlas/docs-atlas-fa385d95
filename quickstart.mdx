## Prerequisites

- Python 3.11 or later
- [Poetry](https://python-poetry.org/docs/#installation) for dependency management

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/tvytlx/ai-agent-deep-dive.git
    cd ai-agent-deep-dive
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    poetry install
    ```
  </Step>
  <Step title="Run the agent">
    ```bash
    poetry run agt "hello"
    ```

    You should see a streaming response from the built-in `FakeLLMClient`:

    ```
    [fake-llm] 你刚才说的是：hello
    ```
  </Step>
</Steps>

## Try the CLI options

**Send a custom prompt:**
```bash
poetry run agt "Please summarize what you can do"
```

**List discovered skills from a directory:**
```bash
poetry run agt --skills-dir ./skills --list-skills
```

**Use a custom skills directory:**
```bash
poetry run agt --skills-dir /path/to/your/skills "Start task"
```

<Note>
The current implementation uses `FakeLLMClient`, which echoes your input back in a streaming fashion. This lets you test the agent loop without a real API key. See [Python implementation notes](/implementation/python-notes) for how to swap in a real LLM.
</Note>

## What just happened?

When you run `agt "hello"`, the following sequence executes:

1. `cli.py` parses arguments and creates an `Agent` instance
2. The `echo` tool is registered
3. Any skills in `--skills-dir` are discovered via `SKILL.md` files
4. `agent.run("hello")` starts the main loop
5. The agent calls `FakeLLMClient.stream_text()` and collects chunks
6. The response is printed and the loop ends

This is the core agent loop — the same pattern scales up to real LLMs, tool calls, and multi-turn tasks.
